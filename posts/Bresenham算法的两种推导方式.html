            • Bresenham算法的两种推导方式

            1. 问题描述

            对于直线绘制时，确定最大步进方向后，另一个方向何时步进，我们可以用Bresenham算法进行解决。在以下推导中，我们假定最大步进方向始终为X轴方向，起点位于左下，终点位于右上。

            2. 中点推导

            对于一般如下的直线公式

            (1)f(x,y)=y−kx−bk=ΔyΔx

            我们记起点为(x0,y0)，那么对应上式有

            (2)f(x0,y0)=y0−kx0−b=0

            当起点绘制结束后，X轴方向进行步进，而对于Y轴方向，我们不确定是否需要步进，不妨考虑下两个候选点的中点

            (3)x1=x0+1y1=y0+0.5

            此时我们考虑该中点的值，记为d0

            (4)d0=f(x1,y1)=f(x0+1,y0+0.5)=y0+0.5−k(x0+1)−b=0.5−k+f(x0,y0)=0.5−k

            若中点值d0大于0，则说明直线处于中点下方，更靠近下方的点，选取的下一个点应为右侧的点，Y轴方向不步进

            (5)d0>0:(x1,y1)=(x0+1,y0)

            对应的下一个需要考虑的中点为X轴步进1步，Y轴步进0.5步，对应中点值为d1如下

            (6)d1=f(x2,y2)=f(x1+1,y1+0.5)=f(x0+2,y0+0.5)=y0+0.5−k(x0+2)−b=0.5−2k+f(x0,y0)=d0−k

            反之，若中点值小于0(等于0时两者皆可，我们考虑上方的点)，则说明直线处于中点上方，更靠近上方的点，Y轴方向也需要步进

            (7)d0≤0:(x1,y1)=(x0+1,y0+1)

            此时下一个中点值d1应如下

            (8)d1=f(x2,y2)=f(x1+1,y1+0.5)=f(x0+2,y0+1.5)=y0+1.5−k(x0+2)−b=1.5−2k+f(x0,y0)=d0+1−k

            综合(8)，(9)，(10)，关于中点值d我们有如下推论

            (9)d0=0.5−kdn+1={dn−k,dn>0dn+1−k,dn≤0yn+1={yn,dn>0yn+1,dn≤0

            对应实际代码中，我们再考虑上面的递推关系，不难发现其中的0.5与k都是需要进行除法运算，而我们始终只要进行考虑中点值的正负，因此可以考虑根据直线函数，公式（1）将中点值放大，记作D

            (10)D=d∗2∗Δk

            这样更新后，有以下递推关系

            (11)D0=1−2ΔyDn+1={Dn−2Δy,dn>0Dn+2Δx−2Δy,dn≤0yn+1={yn,Dn>0yn+1,Dn≤0

            对应代码如下

            xxxxxxxxxx
                int y = ay;
                int d = 1 - 2 * abs(by - ay);//D0 = 1 - 2*dy
                for (int x = ax; x <= bx; x++)
                {
                    if (steep) // 绘制
                    {
                        framebuffer.set(y, x, color);
                    }
                    else
                    {
                        framebuffer.set(x, y, color);
                    }
                    d -= 2 * abs(by - ay);//D0 > 0时,D1 = D0 - 2dy, y方向不变
                    if (d < 0)//D0 < 0时,D1 = D0 + 2dx - 2dy,y方向上进一
                    {
                        y += (by > ay ? 1 : -1);
                        d += 2 * bx - ax;
                    }
                }

            3. 偏差值推导

            如果考虑Y轴上实际的步进方向，那么根据直线函数，公式（1），有如下递推关系

            (12)xn+1=xn+1yn+1=yn+k

            可以看到，Y轴上每次的步进大小均为k，如果用d表示实际交点在Y轴上的偏差，则有

            (13)d0=0, dn+1=dn+k

            考虑从起点出发后的第一个点

            (14)(x0,y0)→(x1,y1)=(x0+1,y0+d1)=(x0+1,y0+k)

            当d>0.5时，说明交点偏上，应取上一个点，反之取下面的点

            (15)y1={y0+1,d1>0.5y0,d1≤0.5

            但当我们考虑再下一个点时，会发现，若d>0.5时，则在后续递增，d会始终大于0.5，每一步都会在Y轴上递进，此时我们需要将d进行修正，最终d的递推关系如下

            (16)d0=0dn+1={dn+k−1,dn>0.5dn+k,dn≤0.5

            对应实际代码中，我们同样可以对d进行调整，从而优化到整数计算，同样将偏差值放大2*dx，记作D，此时

            (17)D0=0Dn+1={Dn+2Δy−2Δx,Dn>ΔxDn+2Δy,Dn≤Δxyn+1={yn+1,Dn>Δxyn,Dn≤Δx

            对应代码如下

            xxxxxxxxxx
                int y = ay;
                int d = 0;//考虑Y轴上的偏差值D
                for (int x = ax; x <= bx; x++)
                {
                    if (steep)//绘制
                    {
                        framebuffer.set(y, x, color);
                    }
                    else
                    {
                        framebuffer.set(x, y, color);
                    }
                    d += 2 * abs(by - ay);
                    if (d > bx - by)//D > dx，此时需要步进
                    {
                        y += (by > ay ? 1 : -1);
                        d -= 2 * (bx - ax);//步进后对偏差值作修正
                    }
                }

            实际开发过程中，我们还可以将dx移项，记作E，使得偏差值最终与0比较大小，参考如下

            (18)E0=−ΔxEn+1={En+2Δy−2Δx,En>0En+2Δy,En≤0yn+1={yn+1,En>0yn,En≤0

            对应代码如下

            xxxxxxxxxx
                int y = ay;
                int e = -1 * (bx - ax);
                for (int x = ax; x <= bx; x++)
                {
                    if (steep)//绘制
                    {
                        framebuffer.set(y, x, color);
                    }
                    else
                    {
                        framebuffer.set(x, y, color);
                    }
                    e += 2 * abs(by - ay);
                    if (e > 0)
                    {
                        y += (by > ay ? 1 : -1);
                        e -= 2 * (bx - ax);
                    }
                }
